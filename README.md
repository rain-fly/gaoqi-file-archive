# gaoqi-file-archive

> 基于 Ollama 视觉模型的 Claude Code Skill - 高新技术企业材料自动分类与整理

[![Python 3.8+](https://img.shields.io/badge/Python-3.8+-blue.svg)](https://www.python.org/)
[![Ollama](https://img.shields.io/badge/Ollama-Local%20AI-green.svg)](https://ollama.com/)

## 痛点问题

整理高企认定材料时，你是否遇到过这些困扰？

| 传统方式 | 使用本工具 |
|---------|-----------|
| 文件名混乱如 `新建文件夹(1).pdf`、`微信图片_20240315.jpg` | 自动识别内容并重命名为 `CERT_001_国药证字Z20050605.pdf` |
| 手工分类 500+ 文件需要整整一天 | AI 自动分类，几分钟完成 |
| 同一产品证书分散在多个文件夹 | 按类型归档，同类产品集中管理 |
| 人工核对容易遗漏关键材料 | 分类报告完整记录，便于审核 |

## 核心功能

- **智能识别** - 基于视觉大模型，自动识别图片/文档内容
- **8类标准分类** - 严格按照高企认定评审要求分类
- **自动重命名** - 按规则统一命名，便于归档检索
- **增量处理** - 支持断点续传，不重复处理已分类文件
- **分类报告** - 生成完整处理日志，支持审计追溯

## 支持的材料类型

| 类型 | 说明 | 示例文件 |
|------|------|---------|
| `IMG` | 产品图片 | 产品外观照、生产线实拍 |
| `CERT` | 产品证书 | ISO认证、3C证书、资质证明 |
| `LIC` | 生产批文 | 生产许可证、药品注册批件 |
| `PROC` | 工艺亮点 | 工艺流程图、配方技术参数 |
| `TEST` | 检测报告 | 质检报告、三年内的检测结果 |
| `CONT` | 销售合同 | 购销合同、采购协议 |
| `INVO` | 销售发票 | 增值税发票、专用发票 |
| `REPO` | 用户报告 | 使用反馈、客户体验报告 |

## 快速开始

### 环境要求

- Python 3.8+
- [Ollama](https://ollama.com/) 本地部署
- 推荐模型：`qwen3.5vl` 或 `llava`

### 安装

```bash
# 克隆项目
git clone <repository-url>
cd 高企智能体测试

# 安装依赖
pip install -r requirements.txt

# 拉取 Ollama 模型
ollama pull qwen3.5vl
```

### 使用

```bash
# 基本用法
python scripts/pipeline.py -i ./客户原始数据 -o ./整理结果

# 使用更大的模型
python scripts/pipeline.py -i ./客户原始数据 -o ./整理结果 --model qwen3.5vl

# 移动文件而非复制（节省空间）
python scripts/pipeline.py -i ./客户原始数据 -o ./整理结果 --move
```

## 文件命名规则

整理后的文件按以下规则命名：

```
{类型缩写}_{序号}_{发票号/产品信息}.{扩展名}
```

**示例：**

| 原始文件名 | 整理后 |
|-----------|--------|
| `新建文件夹(1).pdf` | `CERT_001_国药证字Z20050605_阿胶养血颗粒.pdf` |
| `微信图片_20240315.jpg` | `IMG_001_阿胶养血颗粒.jpg` |
| `合同_abc_2024.pdf` | `CONT_001_XS202501014_强力枇杷露225ml60瓶件.pdf` |
| `发票_xxx.pdf` | `INVO_001_24342000000062288367_中成药半夏糖浆-中成药橘红颗粒-中成药六味地黄丸.pdf` |


## 配置选项

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--input`, `-i` | 必填 | 原始材料目录 |
| `--output`, `-o` | 必填 | 整理结果目录 |
| `--ollama-url` | http://localhost:11434 | Ollama 服务地址 |
| `--model` | qwen3.5vl | 使用的模型 |
| `--no-think` | False | 禁用思考模式（更快） |
| `--move` | False | 移动文件（默认复制） |
| `--no-skip-existing` | False | 禁用增量模式 |

## 常见问题

**Q: 分类结果不准确怎么办？**

A: 可以在 `pipeline.py` 中调整 `keyword_fallback` 关键词规则，或使用更强大的模型如 `qwen3.5vl`。

**Q: 支持哪些文件格式？**

A: 目前支持 `.pdf`, `.docx`, `.jpg`, `.jpeg`, `.png`, `.bmp`, `.tiff`, `.gif`。

**Q: 如何增量处理新文件？**

A: 默认启用增量模式，已处理过的文件会自动跳过。

**Q: 可以不使用 Ollama 吗？**

A: 当前版本依赖 Ollama 视觉模型进行内容识别，暂不支持其他后端。
